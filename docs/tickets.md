# 券系统

[返回目录](README.md)

## 强化券

`ticket_a`，显示名为“强化券”。

作用：打开 GUI，让玩家选择要强化的武器词条。成功率随使用次数递减。失败会消耗强化券，并为下次普通强化增加 3%-13% 成功率。

| 已使用次数 | 基础成功率 |
| ---: | ---: |
| 0 | 100% |
| 1 | 100% |
| 2 | 90% |
| 3 | 80% |
| 4 | 65% |
| 5 | 50% |
| 6 | 35% |
| 7 | 20% |
| 8 | 10% |
| 9 | 5% |
| 10+ | 从 10% 开始按 `0.10 * 0.5^(useCount - 8)` 继续大幅衰减，最低 1% |

### 品质修正

普通强化券会按武器品质修正成功率和数值强化量。超级强化券仍然必定成功，但强化量仍按品质区间计算。

| 品质 | 成功率倍率 | 强化倍率区间 |
| --- | ---: | --- |
| common | 1.10x | 1.20x - 2.70x |
| rare | 1.00x | 1.15x - 2.50x |
| epic | 0.90x | 1.10x - 2.30x |
| legendary | 0.75x | 1.05x - 2.10x |
| special | 1.00x | 1.10x - 2.50x |

强化倍率在区间内按 `min + (max - min) * random^2` 计算，因此更偏向低值。

整数/等级类词条保持固定增长，例如连锁目标 +1、亢奋最高 3、用不坏最高 5、伤害储存次数减少最高 15。

## 超级强化券

`super_ticket_a`，显示名为“超级强化券”。

作用：打开同样的强化选择 GUI，但强化必定成功。

## 开发券

`ticket_b`，显示名为“开发券”。

| 目标 | 效果 |
| --- | --- |
| 非 Roguelike 普通物品 | 开发为 special_weapon 特殊品质武器，保留原版附魔，并保留原物品伤害、攻击速度和已有攻击距离属性 |
| Roguelike 武器 | 打开 GUI 确认后随机添加一个新武器词条 |
| 防具 | 代码支持打开 GUI 并尝试随机添加防具词条；当前没有注册防具随机词条，因此不会产生可选词条 |

Roguelike 武器不再限制开发券添加随机词条的槽位数量。只要当前武器仍有未拥有的可用武器词条，就可以继续使用开发券添加；已经拥有的同类词条不会重复添加。

## 工具开发券

`tool_ticket_b`，显示名为“工具开发券”。

作用：只能用于 Roguelike 镐或斧，只会从工具限定词条池中随机添加词条。当前工具限定词条包括 `ore_highlight`。

## 移除券

`ticket_c`，显示名为“移除券”。

| 目标 | 效果 |
| --- | --- |
| Roguelike 武器 | 打开 GUI 选择一个插件词条移除，并为下次普通强化券成功率增加 10% |
| 防具 | 代码支持移除已记录的插件防具词条；当前没有注册防具随机词条，通常不会有可移除的插件防具词条 |

## 当前防具词条限制

防具词条系统的数据结构和 GUI 流程已经存在，但 `ArmorAffixManager` 当前没有注册任何随机防具词条。实际服内表现以 [词条数据](affixes.md) 中的防具词条说明为准。
